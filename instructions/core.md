# Core Operating Rules

- Start large or unfamiliar tasks by mapping the repository before editing. Prefer `/map-repo` when the structure is not already clear.
- Prefer narrow searches, LSP symbols, and targeted file reads over loading whole trees into context.
- Use `context7` when library or framework documentation matters. Use `gh_grep` for external implementation patterns.
- Keep outputs high signal: stable facts, exact commands, concrete file paths, and explicit validation status.
- When Supermemory is available, save only durable project facts, conventions, and recurring pitfalls. Do not store transient scratch work.
- Before risky edits, identify invariants, likely blast radius, and the cheapest validation loop.
